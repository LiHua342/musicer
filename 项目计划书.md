# 🎵 Musicer - 音乐播放器项目计划书

---

## 一、项目概述

基于原生 **HTML + CSS + JavaScript** 构建一个功能完善的浏览器端音乐播放器，无需任何第三方框架或库，支持本地音乐播放、播放列表管理和本地音乐文件导入。

---

## 二、技术架构

```
┌─────────────────────────────────────────────────┐
│                   index.html                      │
│              (页面结构 / DOM 骨架)                 │
├─────────────────────────────────────────────────┤
│                   css/style.css                   │
│              (样式 / 动画 / 响应式)                │
├──────────────────┬──────────────────────────────┤
│  js/player.js     │      js/playlist.js           │
│   (音频核心引擎)   │      (播放列表管理)            │
├──────────────────┴──────────────────────────────┤
│                  js/ui.js                         │
│     (界面渲染 / 事件绑定 / 用户交互)               │
├─────────────────────────────────────────────────┤
│                  js/main.js                       │
│         (应用入口 / 模块初始化与协调)               │
└─────────────────────────────────────────────────┘
```

### 核心技术选择

| 层级 | 技术 | 说明 |
|------|------|------|
| 音频播放 | HTML5 `<audio>` / Web Audio API | 核心播放能力，支持 MP3/WAV/OGG/FLAC 等格式 |
| 文件读取 | FileReader API + `<input type="file">` | 导入本地音乐文件 |
| 数据存储 | localStorage | 持久化播放列表（存储文件名和 Base64 数据） |
| 界面交互 | DOM 事件 + CSS3 动画 | 原生交互，无框架依赖 |
| 元数据解析 | 基础解析（文件名提取） | 从文件名推断歌曲名和歌手（如 `歌手 - 歌名.mp3`） |

## 三、功能模块详设

### 模块 1：音频播放核心 (js/player.js)

| 功能点 | 描述 |
|--------|------|
| `play()` | 播放当前曲目 |
| `pause()` | 暂停播放 |
| `togglePlay()` | 播放/暂停切换 |
| `next()` | 切换到下一首 |
| `previous()` | 切换到上一首 |
| `seek(time)` | 跳转到指定时间点 |
| `setVolume(level)` | 设置音量 (0-1) |
| `getCurrentTime()` | 获取当前播放进度 |
| `getDuration()` | 获取总时长 |
| `onTimeUpdate(callback)` | 进度更新回调 |
| `onEnded(callback)` | 播放结束回调（自动切歌） |

### 模块 2：播放列表管理 (js/playlist.js)

| 功能点 | 描述 |
|--------|------|
| `add(files)` | 批量添加音乐文件到列表 |
| `remove(index)` | 移除指定歌曲 |
| `clear()` | 清空播放列表 |
| `getList()` | 获取当前播放列表 |
| `getCurrent()` | 获取当前播放的歌曲信息 |
| `setCurrent(index)` | 设置当前播放索引 |
| `shuffle()` | 随机打乱播放列表 |
| `saveToStorage()` | 持久化到 localStorage |
| `loadFromStorage()` | 从 localStorage 恢复 |

**歌曲数据结构：**
```js
{
  id: string,        // 唯一标识（时间戳+随机数）
  title: string,     // 歌曲名
  artist: string,    // 歌手名
  duration: number,  // 时长（秒）
  blobUrl: string,   // Blob URL（内存中的音频地址）
  fileName: string   // 原始文件名
}
```

### 模块 3：界面渲染与交互 (js/ui.js)

| 功能点 | 描述 |
|--------|------|
| 播放控制栏 | 上一首 / 播放暂停 / 下一首 按钮 |
| 进度条 | 可拖拽的进度条 + 当前时间/总时长显示 |
| 音量滑块 | 音量调节 + 静音切换 |
| 播放列表面板 | 歌曲列表展示，点击切换，当前高亮 |
| 导入按钮 | 触发文件选择对话框 |
| 当前歌曲信息 | 显示歌名、歌手、封面（默认图标） |
| 播放模式切换 | 列表循环 / 单曲循环 / 随机播放 |
| 拖拽导入 | 支持拖拽文件到播放器区域导入 |
| 空状态提示 | 无歌曲时的引导提示 |

### 模块 4：应用入口 (js/main.js)

- 初始化各模块
- 协调模块间通信
- 绑定全局事件
- 从 localStorage 恢复状态

## 七、关键实现细节

1. **音乐导入**：通过 `<input type="file" accept="audio/*" multiple>` + 拖拽事件，使用 `URL.createObjectURL()` 生成 Blob URL 供 `<audio>` 播放。
2. **持久化**：因 Blob URL 刷新后失效，需将文件内容以 Base64 存入 localStorage，下次加载时还原。
3. **元数据提取**：优先从文件名解析（格式：`歌手 - 歌名.扩展名`），后续可扩展 ID3 标签解析。
4. **自动切歌**：监听 `audio.onended` 事件，根据播放模式（循环/单曲/随机）决定下一首。
